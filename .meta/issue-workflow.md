# Issue Workflow

## Purpose
Maintain consistency for technical notes and follow-up discussions.

## Rules
1. Create a new issue for a new technical topic.
2. Add a comment to an existing issue for follow-up questions, clarifications, or small extensions on the same topic.
3. Update the main issue body only when the core explanation should be revised or promoted from comments.
4. Keep issue bodies as structured technical notes rather than conversation logs.
5. Keep English and Japanese sections aligned in meaning when both are included.
6. Use Q&A format for follow-up comments when appropriate.
